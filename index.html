GraphQL Reconnaissance & Attack Surface Report

KISEC Project 1 • April 2025

1. Executive Summary

This report summarizes findings from a security assessment targeting a GraphQL endpoint. We identified weaknesses such as schema introspection exposure, unauthenticated WebSocket access, and verbose error disclosure.

2. Objectives

  • Identify exposed GraphQL endpoints
  • Enumerate schema types and operations
  • Detect WebSocket upgrade vulnerabilities
  • Map the GraphQL attack surface

3. Methodology

Using tools like Nmap, ffuf, and curl, we manually and automatically probed the exposed ngrok-based GraphQL instance. Screenshots and logs were collected throughout the process.

4. Key Findings

  • Introspection Enabled: Schema enumeration possible
  • WebSocket Subscriptions: Unauthenticated 101 response
  • Error Disclosure: Message suggests no query validation
  • Open POST Access: Accepts invalid GraphQL queries

5. Risk Evaluation

Risk Impact Likelihood Severity
Introspection Enabled Medium High High
WebSocket Access High Medium High
Verbose Errors Medium High Medium

6. Recommendations

  1. Disable introspection in production
  2. Enforce authentication on WebSocket upgrades
  3. Implement WAF and API rate-limiting
  4. Suppress detailed error messages
  5. Use schema validation and deny malformed queries
